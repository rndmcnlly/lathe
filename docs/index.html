Lathe

Code interpreter meets coding agent — in your own chat

GitHub Install JSON
ChatGPT has code interpreter. Claude has artifacts and computer use. Lathe brings the same class of capability to Open WebUI — with any model you choose, on infrastructure you control. One Python file, installed once by an admin. After that, every model on the server gets a full coding agent toolkit backed by a persistent cloud sandbox: shell, file editing, code search, web services, SSH, and a browser IDE. One sandbox per user, survives across conversations.
Why

The tools you expect, where you already work

Proprietary chat platforms bundle their coding agent capabilities behind closed ecosystems. Terminal-native agents like Claude Code and Codex are powerful but leave non-developers behind. Lathe occupies the space between: the conversational interface of a web chat with the full sandbox capabilities of a coding agent.

Like ChatGPT Code Interpreter

The model writes and runs code in a sandbox, right inside your chat conversation. No terminal needed. Users just talk.

Like Claude Code / Codex CLI

Full Linux VM with bash, file editing, git, package managers, and persistent state across sessions. Not a toy — a real machine.

Your models, your server

Works with any model your Open WebUI instance can reach — open-weight or proprietary. No vendor lock-in on the intelligence.

Multi-user from day one

Every user gets their own isolated sandbox. Admins install once; users don't configure anything. Built for shared deployments.

Tools

What your agent can do

Ten tools, same as the best proprietary agents. Each one runs against a persistent Linux sandbox that follows you anywhere your browser does.

lathe

Built-in manual. The model calls this to learn the sandbox model, workflows, and gotchas before its first real tool use.

bash

Run any shell command. Install packages, compile code, run tests, manage git repos.

read

Read files with line numbers. Supports offset/limit for large files.

write

Create or overwrite files. Parent directories are created automatically.

edit

Exact string replacement in files. Safer than rewriting the whole file.

glob

Search for files by pattern. Returns a hierarchical listing that collapses dense directories.

grep

Search file contents by regex. Returns matches grouped by file with line numbers.

expose

One-step file browser ("dufs"), browser IDE ("code-server"), public URL for any web service ("http:<port>"), or time-limited SSH command ("ssh").

onboard

Load project context and agent skills from your repo's conventions.

destroy

Wipe your sandbox VM and start fresh. Persistent volume data survives. Requires explicit confirmation.

Recipes

Things to try

Real prompts you can send to a model with Lathe enabled. The model figures out which tools to call.

  • Build a web app "Build me a simple todo app with FastAPI and HTMX, then expose it so I can try it."
  • Browse & transfer files "Set up a file browser" — one tool call installs dufs, starts it, and gives you a URL with drag-and-drop upload/download.
  • Work on a repo "Clone my repo, run the tests, and fix any failures."
  • Provide credentials Set env_vars in your user settings (e.g. {"GITHUB_TOKEN":"ghp_..."}) — every bash command gets them, the model never sees the values.
  • Go hands-on "Give me SSH access" — get a command you can paste into your terminal or VS Code.
  • Full IDE in the browser "Set up code-server" — one tool call installs code-server and gives you VS Code in a browser tab.
Persistence

Your workspace survives

ChatGPT's code interpreter resets every conversation. Lathe doesn't. Your sandbox is yours — files, installed packages, git repos, running servers. Start a conversation Monday, come back Wednesday, everything is where you left it.

The sandbox stops after ~15 minutes of idle, then archives about an hour later. Either way, the next tool call wakes it transparently — your files and installed packages survive both. Only running processes are lost. If you want a clean slate, ask the model to destroy it.

Configuration

User settings

Most users need zero configuration. But if you work with APIs or private repos, you can set environment variables that get injected into every shell command:

# In Open WebUI → Settings → Tools → Lathe → env_vars: {"GITHUB_TOKEN": "ghp_...", "OPENAI_API_KEY": "sk-..."}

Values are never shown to the model. They're exported in the shell preamble with single-quoted values to prevent expansion.

Deployment

For admins

Lathe is a single Python file (lathe.py) deployed as an Open WebUI Tool. It needs a Daytona API key and a deployment label. Configure these as admin Valves after install.
